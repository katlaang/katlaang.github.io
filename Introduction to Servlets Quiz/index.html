Introduction to Servlets Quiz


Question 1: What is the difference between a web server and a web container?

Ans: Web Server deals with the http request and response. but the web container deals with the Servlets and their life cycle. Request of any user first comes to the Web Server and then Web Server sends that request to the Web Container.

Question 2: What is a servlet?

Ans: Servlet is a java class with capabilities of server which handles requests from the client with the help of Web Container and Web Server. A java class extends HttpServlet. The HttpServlet class provides methods, such as doGet and doPost, for handling HTTP-specific services.

Question 3: How do web servers and web containers interact with servlets?

Ans: Whenever a user request or want any response. Web server accepts that request and transfer that to Web Container and then Web Container deals with the Servlets life cycle. Example of Web Sever and Web container. Apache is Web Server and Tomcat is the Web container.

Question 4: Who creates request objects?

Ans:The servlet container locates the servlet, creates request and response objects and passes them to the servlet,and returns to the web server the response stream that the servlet produces.

Question 5: What are the states in the servlet lifecycle?

Ans:
Step 1: Loading of Servlet.
Step 2: Creating instance of Servlet.
Step 3: Invoking initMethod
Step 4: Invoking Service Method
Step 5: Invoking Destroy method.

Question 6: Who calls init and when?

Ans: init method is called by the container. It is called when the servlet is first created, and not called again for each user request

Question 7: Which of init, service, and doGet should you override?

Ans: We override doGet method.

Question 8: In what sense are servlets multi-threaded?

Ans:There is no such interface as MultiThreadedModel. Servlets are intrinsically multithreaded. This means a single instance can be accessed by more than one thread.

Question 9: What are the implications of this for servlet instance variables?

Ans: Servlet instance variables should implement appropriate concurrency.
